# Claude 명령어: Project Error Check

프로젝트에서 발생하는 에러를 빠르게 감지합니다. 첫 번째 에러를 발견하면 즉시 멈추고 해당 에러 정보만 보여줍니다.

## 사용법

```
/project-error-check
```

## 수행 내용

1. **TypeScript 타입 체크**: `tsc --noEmit`으로 타입 에러 감지
2. **IDE 진단 정보**: VS Code 진단 도구로 모든 파일의 에러 수집
3. **첫 에러에서 멈춤**: 에러 발견 즉시 중단, 모든 에러를 찾지 않음
4. **수정 없음**: 에러만 보고, 수정은 수행하지 않음

## 에러 체크 순서

1. TypeScript 컴파일 에러 (가장 빠름)
2. IDE 진단 에러 (파일별 상세 정보)
3. 결과 리포트

## 출력 형식

에러가 발견되면:
```
❌ 에러 발견!

파일: src/pages/index.tsx
라인: 42
에러: Property 'foo' does not exist on type 'Props'

[중단됨 - 첫 번째 에러에서 멈춤]
```

에러가 없으면:
```
✅ 에러 없음!

프로젝트에서 감지된 에러가 없습니다.
```

## 주의사항

- 이 커맨드는 **에러만 감지**합니다 (경고/정보는 제외)
- **첫 번째 에러에서 멈추므로** 모든 에러를 찾지 않습니다
- **수정은 수행하지 않습니다** - 에러 리포트만 제공
- 빠른 피드백을 위해 최적화되었습니다

## 사용 시나리오

- 빌드 전 빠른 에러 체크
- 코드 리뷰 전 타입 안정성 확인
- 배포 전 에러 유무 파악
